# 输出模板与标准答法

## 目录
- 使用规则
- 架构设计答法
- Bug 排查答法
- 代码审查答法
- Swift 并发答法
- 性能分析答法
- 重构与迁移路线答法
- 严格输出要求

## 使用规则
- 需要输出方案、审查结论、排障结论、迁移路线、性能分析时，直接套用本文件模板。
- 结论必须先给判断，再给证据，再给修法。
- 禁止只给概念解释，不给执行步骤。

## 1. 架构设计答法
适用于：模块设计、页面重构、网络层设计、状态治理。

输出结构：

```text
结论
- 采用什么架构
- 分层边界如何划分
- 依赖方向如何约束

原因
- 当前问题是什么
- 为什么该方案最合适
- 不采用其他方案的原因

落地方式
- 第一步改什么
- 第二步改什么
- 第三步如何收口

风险
- 会影响哪些模块
- 哪些地方最容易引入回归

验证
- 单元测试怎么补
- 集成验证怎么做
```

## 2. Bug 排查答法
适用于：Crash、状态错乱、布局异常、并发问题、偶现问题。

输出结构：

```text
问题现象
- 用户可见现象
- 触发条件
- 影响范围

证据
- 调用链
- 状态变化
- 生命周期
- 线程 / Actor / Task 上下文

根因
- 真正出错的结构位置
- 为什么在这个时机触发

修复方案
- 结构性修复点
- 为什么不是补丁修复

验证结果
- 如何复现
- 如何证明已修复
- 是否评估副作用
```

## 3. 代码审查答法
适用于：PR Review、方案 Review、重构 Review。

输出结构：

```text
严重问题
1. [问题标题]
   - 位置
   - 原因
   - 风险
   - 修法

中等级问题
1. [问题标题]
   - 原因
   - 影响
   - 修法

验证缺口
- 缺什么测试
- 缺什么验证

结论
- 是否可合入
- 合入前必须完成什么
```

执行要求：
- 严重问题先于风格问题。
- 正确性先于可读性。
- 风险先于偏好。

## 4. Swift 并发答法
适用于：Actor 设计、任务取消、回调迁移、Sendable 审查。

输出结构：

```text
并发模型判断
- 哪些状态属于主隔离域
- 哪些状态属于独立隔离域
- 哪些数据需要 Sendable

当前风险
- 数据竞争点
- 非结构化任务
- 取消失效
- UI 跨线程更新

修复方案
- actor 如何建
- Task 层级如何收敛
- 旧 callback 如何桥接

验证
- 编译期并发检查
- 真机行为验证
- 取消链路验证
```

## 5. 性能分析答法
适用于：启动慢、滚动卡顿、内存上涨、页面刷新过重。

输出结构：

```text
性能问题定义
- 指标是什么
- 发生在什么路径

证据
- Instruments 结果
- 主线程热点
- 内存热点
- 刷新热点

根因
- 具体慢在哪里
- 为什么慢

优化方案
- 删除什么
- 下沉什么
- 异步化什么
- 缓存什么

验证
- 优化前数据
- 优化后数据
- 是否有副作用
```

## 6. 重构与迁移路线答法
适用于：大型遗留模块拆分、UIKit 转 SwiftUI、回调迁移 async/await。

输出结构：

```text
目标
- 这次重构解决什么
- 不解决什么

阶段划分
1. 建抽象
2. 迁调用
3. 删旧实现
4. 补测试

每阶段产物
- 新协议
- 新模块
- 兼容层
- 验证项

风险控制
- 如何避免大面积回归
- 如何灰度或逐步替换
```

## 7. 严格输出要求
- 回答架构问题时，不只讲模式名称，必须讲边界、依赖方向和状态归属。
- 回答 Bug 问题时，不只讲猜测，必须讲证据。
- 回答性能问题时，不只讲优化点，必须讲指标。
- 回答审查问题时，不只讲风格，必须讲风险。
- 回答迁移问题时，不只讲终态，必须讲阶段。
