# 执行剧本

## 使用规则
- 遇到复杂任务时，必须先选择对应剧本，再进入分析和实现。
- 剧本定义的是执行顺序，不是背景知识说明。
- 不得跳过“取证、边界、验证”三步。

## 目录
- 接手遗留页面
- 排查偶现 Crash
- 做一次性能优化
- 做一次并发迁移
- 做一次大型重构

## 接手遗留页面
适用于：
- 超大 ViewController
- 状态散落页面
- UIKit / SwiftUI 混合老页面

执行顺序：
1. 定义页面边界：它负责什么，不负责什么。
2. 识别状态来源：本地状态、远端状态、缓存状态、导航状态。
3. 标出越界代码：网络、路由、缓存、埋点、权限、格式化。
4. 建最小重构目标：先拆状态、再拆依赖、最后拆结构。
5. 明确迁移阶段：不允许一次性大爆炸重构。
6. 补测试和回归路径。

交付要求：
- 给出页面问题分层图
- 给出阶段化改造顺序
- 给出回归验证范围

## 排查偶现 Crash
适用于：
- 难复现崩溃
- 线上偶发异常
- 状态错乱引发的随机问题

执行顺序：
1. 定义现象：崩溃点、频率、设备、系统版本、触发条件。
2. 建证据链：日志、调用栈、状态流、生命周期、线程/Actor。
3. 区分崩溃点与根因。
4. 沿输入源 -> 数据转换 -> 状态管理 -> 并发边界 -> 生命周期 -> UI 渲染回溯。
5. 做结构性修复，不做延迟、重试、判空补丁。
6. 给出修复验证闭环和副作用评估。

交付要求：
- 给出根因结论
- 给出修复前后证据
- 给出复现与回归验证路径

## 做一次性能优化
适用于：
- 启动慢
- 列表卡顿
- 页面刷新重
- 内存异常增长

执行顺序：
1. 明确指标：启动时长、FPS、主线程耗时、内存峰值、CPU。
2. 锁定路径：冷启动、热启动、首屏、滚动、切换页面、后台切前台。
3. 用工具取证：Time Profiler、Core Animation、Memory Graph、MetricKit。
4. 找出最重热点，不同时处理多条主因。
5. 明确优化动作：删除、下沉、异步化、缓存、瘦身。
6. 对比优化前后数据，评估正确性和体验是否回归。

交付要求：
- 给出基线
- 给出热点
- 给出优化前后对比

## 做一次并发迁移
适用于：
- callback 迁 async/await
- GCD 迁结构化并发
- 串行队列迁 actor 隔离

执行顺序：
1. 列出当前并发模型：谁创建任务，谁写状态，谁切主线程。
2. 列出共享可变状态和跨域传递数据。
3. 先设计隔离域，再选 `@MainActor`、`actor`、`TaskGroup`、`async let`。
4. 桥接旧接口时保证只 resume 一次。
5. 建取消链路，阻止过期结果回写。
6. 用编译检查、真机行为、取消验证确认迁移成功。

交付要求：
- 给出隔离模型图
- 给出迁移顺序
- 给出取消与回写验证

## 做一次大型重构
适用于：
- 模块拆分
- 导航重建
- 状态模型重建
- 网络层重构

执行顺序：
1. 定义重构目标和明确不做的范围。
2. 写决策记录，比较候选方案。
3. 划分阶段：建抽象、迁调用、删旧实现、补测试。
4. 识别高风险模块和回滚点。
5. 每阶段做行为一致性验证。
6. 最后再清理历史兼容层。

交付要求：
- 给出决策记录
- 给出阶段计划
- 给出每阶段验证方法
